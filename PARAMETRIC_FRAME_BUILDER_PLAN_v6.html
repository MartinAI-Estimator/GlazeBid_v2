01What this is
Parametric Frame Builder is the third core module of the GlazeBid ecosystem. It lives as a sidebar tab inside Builder — the same pattern as Studio. Its job is to transform raw frame dimensions into a complete, fabrication-ready engineering package — bill of materials, cut lists, glass schedule, metal material list, door schedule, structural analysis, and shop drawings — using real vendor part numbers and rules-based accessory calculations.

02UX philosophy — three non-negotiables
Visual immediacy
Every input produces an immediate visual response in the CAD canvas. No calculate button. No update preview. The frame is alive. Type a width, the frame redraws. Change bay count, a mullion appears. Adjust a panel, adjacent panels redistribute. The BOM updates in the same moment.
Estimators think visually
Everything built in Frame Builder is represented visually at all times. The CAD workspace is the source of truth the estimator trusts. Numbers in fields are confirmed by what they see on screen. If the visual is wrong, the inputs are wrong.
Sidebar native — no context switching
Frame Builder lives inside Builder as a sidebar tab. The estimator builds frames and the bid sheet prices them in real time. No export step, no transfer, no sync to wait for. Finishing a frame and clicking the Bid Sheet tab shows the line items immediately.

05Architecture decisions — locked
Locked
Two-layer data model. Archetype (geometry truth, system-agnostic) + Vendor mapping (resolves member roles to real part numbers). BOM computed once, resolved N times for alternates.
Locked
Frame Builder = sidebar tab in Builder. Not a third Electron window. Lives inside Builder like Studio. Sidebar resizable. Direct store write — no IPC relay for data transfer to bid sheet.
Locked
Visual immediacy is non-negotiable. Every input field triggers an immediate CAD canvas re-render and BOM update. No calculate button anywhere in the application.
Locked
Frame groups standard, elevation toggle optional. System, finish, and glass assigned at group level and cascade. Elevation grouping off by default, on when the job warrants it.
Locked
Cut optimizer scrap thresholds. Under 8% = green. 8–10% = yellow warning. Over 10% = red review flag. Applied per part number in the metal material list.
Locked
Sill height = AFF, available at bid time. Entered on Identity tab. Feeds CAD environment, structural anchor spacing, tempering auto-flag, and high-base advisory.
Locked
All-glass panel layout. Equal default. Auto-redistribute on every adjustment. LINKED groups when panels share the same size. Door panels LOCKED and upstream of the pool. Minimum panel width = 4".
Locked
All-glass hardware vendors. CRL, Dorma, ASSA ABLOY, Blumcraft. One vendor per job when possible. Same two-layer archetype + vendor mapping model as framed systems.
Locked
Custom conditions are always manual. High bases, corners, specialty trim — manual add. Soft advisory shown when sill is at or near grade. No rule engine guessing design intent.
Locked
Alternate summary is customer-facing. Title block, company logo, professional deliverable language. Part of the bid package submitted to the GC.
Locked
Dashboard lives in both Frame Builder and Builder. Frame Builder view: frames, scrap, structural flags, spec compliance. Builder view adds cost, margin, and labor cost.
Locked
Vendor data entry. Seeded library for 3 vendors + CW counterparts. Price book PDF upload with AI extraction for all others. Custom parts library per company for non-catalog items.
Dropped
Project templates. Every project is unique. Templates create false head starts. Not building this.
Dropped
Installation sequence, submittal log, RFI generator, hardware finish checker, lead time flags, field verification checklist. Good ideas, not needed for the initial build. Can be added post-launch.

08Build sequence — 8 phases
Phase 1Data schema, archetypes, fabrication rules engineFoundation
Core types
  • Archetype, VendorSystem, MemberMapping, FabricationRules, BOMOutput
  • FrameIdentity: mark, group, elevation, class, scope, AFF, mock-up, qty, notes
  • FrameGeometryType enum: RECT, RAKED, ARCHED, ARCH_FULL, CIRCLE, TRIANGLE, POLYGON, RIBBON
  • AllGlassPanelState: EQUAL, LINKED, LOCKED
  • FrameContext: wall type, AFF datum, head condition, adjacent frame IDs
  • MetalTakeoffRecord, RevisionRecord, FrameEngineeringPackage types
Engine + seed data
  • 10 core archetypes: 5 storefront, 4 curtain wall, 1 window wall
  • Fabrication rules engine — industry defaults
  • BOM resolver: archetype + vendor + dimensions → full BOM
  • Seed: Kawneer 451T, Tubelite T1400, Oldcastle FG-3000 (storefront first)
  • Panel layout engine: EQUAL/LINKED/LOCKED, 4" minimum, auto-redistribute
  • Unit test all calculations against real-world outputs
Phase 2Frame builder UI — CAD workspace, 7-tab input, quick take, visual immediacyPrimary estimator workspace
CAD workspace
  • Sidebar tab integration inside Builder — resizable panel
  • Top-level tab bar: Framed Systems / All-Glass Walls / Glass Takeoff / Metal List / Dashboard
  • Dark CAD workspace — grid, zoom/pan, coordinate readout, scale selector
  • Reactive rendering pipeline — every input triggers immediate canvas re-render
  • Dimension callouts rendered live — string, grid, DLO, AFF labels
  • Layer toggles: frame, glass, dims, context, labels
Input + workflow
  • 7-tab framed input flow with keyboard navigation
  • Quick take mode: fast table entry, group settings auto-applied
  • Estimator notes field on Identity tab
  • Shape type selector, finish selector, joint width, custom DLO override
  • Multi-vendor alternate selector (base + alt 1, alt 2)
  • Frame group assignment and group-level cascade
  • Copy / typical frame function
  • Studio intake: RawTakeoff auto-fills dimensions and identity
  • Caulking calculator auto-runs on BOM resolve

Compliance

  • ADA/IBC door compliance: clear width, vision panel AFF, hardware height
  • Bid unit price calculator: $/LF and $/SF by system class
Phase 5Door builder, operable vents, non-rectangular geometry, vendor seed dataSpecial conditions and schedules
Doors, vents, vendor data

09Open items and risks
Critical
Vendor fabrication rules. Screws per joint, setting block placement, and all accessory rules must come from actual vendor fab manuals. Industry defaults in place but must be replaced before production use.
Medium
Phase 1 test case. Before writing Phase 1 code, define one real frame — vendor, system, dimensions, grid, finish — to validate every formula against known real-world output.
Medium
Price book extraction quality. Vendor PDFs vary in structure. AI extraction needs validation against real price books before trusting output.
Medium
Curtain wall phasing. Prove all architecture on storefront first in Phases 1–3, then extend to CW in Phases 4–6.
Low
IPC contract compatibility. Frame Builder introduces FrameEngineeringPackage and RevisionRecord. Existing RawTakeoff and EngineeredFrame contracts must remain untouched. New channels only.
Out of scope
Not building (initial release): project templates, installation sequence, submittal log, RFI generator, hardware finish checker, lead time flags, field verification checklist, unitized CW, BIM/Revit export, CNC output, thermal compliance simulation, multi-user collaboration.
